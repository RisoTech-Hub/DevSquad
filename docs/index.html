Welcome to devsquad

devsquad is a CLI tool that bridges Slack with Claude Orchestrator multi-agent sessions. It allows you to run AI development teams locally inside tmux sessions while communicating with them directly through Slack.

With devsquad, you can:

  • Create dedicated Slack channels mapped to local tmux sessions
  • Run Claude Orchestrator in the background and interact via Slack
  • Manage multiple AI agent teams working on different projects
  • Integrate with GitHub for task tracking and project management
  • Diagnose and fix issues with built-in health checks

Quick Start

Initialize a new devsquad project inside any local repository. This creates a dedicated Slack channel, starts a background tmux session for the Claude Orchestrator, and wires everything together.

# Install devsquad globally
npm install -g devsquad

# Inside your working directory
devsquad project init --name my-feature
Note

Before running devsquad project init, make sure you have configured your Slack tokens using devsquad config. See the Configuration guide for details.

Key Concepts

Projects

A project is a mapping between:

  • A local directory (your codebase)
  • A tmux session running the Claude Orchestrator
  • A dedicated Slack channel

Daemons

Background processes that bridge Slack events to your local development environment:

  • Listener Daemon: Captures all Slack events via Socket Mode
  • Processor Daemon: Queues messages and injects them into tmux sessions

Agents

The simulated AI team members that collaborate in the tmux session. Each agent has a specific role (Tech Lead, Developer, QA, etc.) and is powered by an underlying model.
