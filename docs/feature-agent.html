Agent Management

Agents are the simulated AI team members that collaborate in the tmux session. Each agent has a specific role (Tech Lead, Developer, QA, etc.) and is powered by an underlying model.

Agent Registry

The agent registry is stored in ~/.devsquad/agents.json and defines:

  • Name: Unique identifier for the agent
  • Role: Team role (Developer, Tech Lead, Architect, etc.)
  • Model: Underlying AI model (e.g., claude-sonnet-4-6)
  • Container: Optional Docker container name for isolation
  • Skills: List of skills for task routing

Default Agents

When you first set up devsquad, you can initialize a default team:

# Initialize with default agents
devsquad agent init

Managing Agents

Command Description
devsquad agent list List all registered agents
devsquad agent show <name> Show full details for an agent
devsquad agent add Register a new agent
devsquad agent remove <name> Remove an agent from the registry

Adding Custom Agents

# Add a new agent with required options
devsquad agent add --name agent-claude-dev --role "Developer" --model claude-sonnet-4-6

# Add agent with container and skills
devsquad agent add --name agent-minimax-dev --role "Developer" --model minimax --container dev-container --skills clean-code,typescript-expert

Agent Roles

devsquad supports several predefined roles for task routing:

  • Developer — Code implementation tasks
  • Tech Lead — Architecture and code review
  • Architect — System design and domain modeling
  • Manager — Project planning and coordination
  • QA Engineer — Testing and quality assurance

Skills

Skills are used to route tasks to the appropriate agent. Each skill represents a specialized capability:

  • clean-code — Code quality and readability
  • typescript-expert — TypeScript type safety
  • code-review-excellence — Code review standards
  • architecture — System structure design
  • domain-driven-design — Business domain modeling
  • planning-with-files — Version-controlled planning
  • concise-planning — Minimal overhead planning
  • testing-patterns — Test structure patterns
  • documentation — Technical documentation
Note

When creating task briefs using devsquad task brief, the appropriate skills are automatically included based on the agent role.
