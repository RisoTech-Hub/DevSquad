Health & Diagnostics

If messages stop flowing or sessions drop, use devsquad doctor to diagnose and repair your devsquad environment. The doctor runs a comprehensive suite of checks to identify and fix common issues.

Running Diagnostics

# Run diagnostics
devsquad doctor

# Run diagnostics for a specific project
devsquad doctor --name my-feature

# Auto-fix issues
devsquad doctor --fix

Diagnostic Checks

The doctor performs checks in three categories:

Global Checks

Check Description
Config File Verifies ~/.devsquad/config.json exists and is valid JSON
Slack Tokens Validates Slack App and Bot tokens are configured
Agents File Verifies ~/.devsquad/agents.json exists
Listener Plist Checks the listener LaunchAgent plist file exists
Listener State Verifies the listener process state
Stale Listeners Detects stale or zombie listener processes

Project Checks

Check Description
Project Exists Verifies project is registered in projects.json
Tmux Session Checks if the tmux session is running
Processor Plist Verifies the processor LaunchAgent plist exists
Processor State Checks if the processor daemon is running

Workspace Checks

Check Description
Git Repository Verifies the workspace is a git repository
Git Remote Checks for configured git remote origin

Auto-Fix Mode

Passing the --fix flag enables automatic repairs:

  • Creates missing config files
  • Kills zombie listener processes
  • Reloads crashed daemons
  • Restores missing plist files
Warning

Auto-fix will modify system state including killing processes and reloading LaunchAgents. Review the output carefully before running with --fix.

Common Issues

Messages not reaching the agent

  1. Run devsquad daemon status to check daemon health
  2. Run devsquad doctor to identify issues
  3. Try devsquad daemon restart

Tmux session not found

  1. Run devsquad project resume --name <project>
  2. Check the project is registered: devsquad project list

Slack authentication errors

  1. Run devsquad config --view to check tokens
  2. Update tokens with devsquad config --bot-token ... --app-token ...
