Architecture

The codebase follows a Layered Architecture with DDD-lite (Domain-Driven Design) principles. It is structured into three main horizontal layers, strictly utilizing domain interfaces for inversion of control.

Layered Diagram

CLI / Commands Layer (src/commands/) User-facing entry points parsing arguments and calling the Application layer.
Application Layer (src/application/) Use cases and orchestrators (e.g., Daemons, Project management).
Domain Layer (src/domain/) Strict interfaces and contracts (zero dependencies).
Infrastructure Layer (src/infra/) External service implementations satisfying the Domain layer (Slack, Redis, Tmux, Docker).
Utils Layer (src/utils/) Configuration files, path helpers, and logging utilities.

Design Patterns

Pattern Where Used Description
Layered / DDD-lite Global Dependencies flow inward towards the Domain. Infrastructure and Application interact through Domain contracts.
Facade SlackService Provides a single entry point for all Slack operations, wrapping both ISlackClient and ISlackSocket.
Adapter SlackBoltSocket, SlackSocketModeAdapter Adapts external libraries like @slack/bolt and @slack/socket-mode into the internal ISlackSocket domain interface.
Queue-based Messaging SlackListenerDaemon → Redis → MessageProcessorDaemon Decouples the receipt of messages from Slack and their injection into tmux sessions via Redis queues.
Watchdog SlackSocketModeAdapter Actively monitors WebSocket heartbeat. Restarts connection if no real message arrives after 5 minutes.
State Machine ProjectStatusService Manages project lifecycle phases: Listening, Planning, Delegating, Waiting, Reporting, Offline, Crashed.
Repository ProjectService Abstracts the project JSON file storage behind a strict CRUD interface.

Background Process Management

Background tasks utilize macOS LaunchAgents (launchd):

  • Listener Daemon: com.devsquad.listener — Runs once globally on the machine.
  • Processor Daemon: com.devsquad.processor.{projectName} — Runs one per registered project.

Each plist config is written into ~/Library/LaunchAgents/. Daemons run without KeepAlive automatically enabled, ensuring devsquad has manual control over startup and shutdown sequences. Standard outputs and errors are written to ~/.devsquad/logs/.
