Project Management

A project in devsquad is the binding agent between your local codebase, a tmux session, and a Slack channel. It serves as the central coordination point for your AI development team.

What Happens During project init

When you run devsquad project init, the CLI performs a multi-step setup:

  1. Creates a Slack channel using the Slack API
  2. Invites required users based on --users flag or DEV_SQUAD_CORE_MEMBERS env var
  3. Generates a unique claudeSessionId for session tracking
  4. Spawns a background tmux session named after the project
  5. Installs and starts a specific MessageProcessorDaemon for this project
  6. Optionally links to GitHub if the gh CLI is authenticated

Project Lifecycle

Initialization

# Initialize a new project
devsquad project init --name my-feature

# With specific users
devsquad project init --name my-feature --users U123,U456

# With GitHub repo
devsquad project init --name my-feature --repo owner/repo

# Skip GitHub integration
devsquad project init --name my-feature --local-only

Stopping a Project

Gracefully stops the Claude session and cleans up resources:

# Stop a project
devsquad project stop --name my-feature

Resuming a Project

Restarts the tmux session and processor daemon:

# Resume a stopped project
devsquad project resume --name my-feature

Project Status Phases

Projects track their current state through phases displayed in Slack:

  • Listening — Waiting for user input
  • Planning — Analyzing the request
  • Delegating — Assigning tasks to agents
  • Waiting — For agent responses
  • Reporting — Summarizing results
  • Offline — Project stopped
  • Crashed — Unexpected error

Gate Modes

Projects can operate in two modes:

  • autonomous — Agents can act without human approval
  • supervised — Requires human approval for certain actions
# Set project mode
devsquad project set-mode --name my-feature --mode autonomous

Managing Projects

Command Description
devsquad project list List all registered projects
devsquad project config --name <project> Show config and state for a project
devsquad project remove --name <project> Remove a project and clean up all resources

Internal Commands

The orchestrator uses these internal commands to update project state:

project update

Updates orchestrator phase and/or current task:

# Update phase
devsquad project update --name my-feature --phase Delegating

# Update task
devsquad project update --name my-feature --task "Implementing login"

# Batch update from stdin
echo '{"phase":"Delegating","task":"...","agents":{"agent-name":"Working"}}' | devsquad project update --name my-feature --batch

project agent

Updates individual agent status:

devsquad project agent --name my-feature --agent agent-claude-dev --status Working
