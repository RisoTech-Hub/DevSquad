Created interactively via devsquad config.

Field Type Description

redis_host string Redis host address (default: 127.0.0.1).

redis_password string Redis password (optional).
slack_status_channel string Channel ID where global agent statuses are broadcasted.
port number Internal HTTP port.
logLevel string Winston log level (e.g., info, debug).

State Files in ~/.devsquad/

File Owner Content / Purpose
projects.json ProjectService Array of configuration objects representing active tracked projects.
daemon-state.json DaemonStatusService Metadata capturing the last shutdown info, leveraged for crash detection.
daemon.shutdown run-listener.ts A temporary flag file written solely on clean, graceful exits.
team-state.json TeamStatusService Cached Docker container statuses for the AI team.
project-{name}.state.json ProjectStatusService Per-project status containing the Slack message TS block to edit on phase updates.
logs/{label}.log LaunchDaemonManager Continuous stdout/stderr streams for each respective daemon process.

macOS LaunchAgent Configuration

LaunchDaemonManager automatically generates .plist files in ~/Library/LaunchAgents/.

  • com.devsquad.listener: Runs the WebSocket receiver.
  • com.devsquad.processor.{projectName}: Dedicated tmux injector instances.

All plists strictly set KeepAlive: false, giving the devsquad CLI absolute authority to restart, load, and manage running Daemons rather than the OS unexpectedly reviving broken processes.
