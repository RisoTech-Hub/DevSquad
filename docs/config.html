Configuration

devsquad relies on several local state files and configuration variables, typically stored in ~/.devsquad/.

Token Configuration

Run devsquad config to securely set your Slack App and Bot tokens. These are required for Socket Mode connections and API interactions.

  • Slack App Token: Starts with xoxa-
  • Slack Bot Token: Starts with xoxb-

Setting Tokens

# Interactive mode (prompts for tokens)
devsquad config

# Non-interactive mode
devsquad config --bot-token xoxb-your-bot-token
devsquad config --app-token xoxa-your-app-token

# View current configuration
devsquad config --view

Config File: ~/.devsquad/config.json

Core global configurations stored in JSON format.

Field Type Default Description
slack_app_token string — Slack Socket Mode App token (starts with xoxa-). Required.
slack_bot_token string — Slack Bot token (starts with xoxb-). Required.
redis_host string 127.0.0.1 Redis host address.
redis_port number 6379 Redis port.
redis_password string — Redis password (optional).
slack_status_channel string general Channel ID for global daemon status broadcasts.
port number 3100 Internal HTTP port.
logLevel string info Winston log level (info, debug, etc.).
protocol_base string — Custom protocol base path (optional).

State Files

These files store persistent state for projects, agents, and daemon information.

~/.devsquad/projects.json

The global registry of all active projects linking directory paths, tmux session names, and Slack channel IDs.

~/.devsquad/agents.json

Definitions of available agents (Developer, Tech Lead, Architect, etc.), mapping roles to underlying models.

Per-Project State: ~/.devsquad/project-{name}.json

Contains the Slack message timestamp for the status block that gets edited on phase updates.

Daemon Logs: ~/.devsquad/logs/

Continuous stdout/stderr streams for each daemon process.

Note

Do not edit these files manually unless necessary. Use the CLI commands (devsquad project add, devsquad agent add) to manipulate them safely.

Environment Variables

Variable Used In Purpose
DEV_SQUAD_CORE_MEMBERS project.ts Comma-separated Slack user IDs to automatically invite to new project channels.

macOS LaunchAgent Configuration

The LaunchDaemonManager automatically generates .plist files in ~/Library/LaunchAgents/:

  • com.devsquad.listener — Runs the WebSocket receiver (single global instance).
  • com.devsquad.processor.{projectName} — Dedicated tmux injector instances (one per project).
Warning

All plists strictly set KeepAlive: false, giving the devsquad CLI absolute authority to restart, load, and manage running daemons rather than the OS unexpectedly reviving broken processes.
