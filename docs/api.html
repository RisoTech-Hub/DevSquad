Domain Interfaces

The application relies exclusively on these strict contracts to perform I/O operations.

ISlackClient

Core Slack REST API abstraction for sending messages and managing channels.

interface ISlackClient {
  postMessage(options: PostMessageOptions): Promise<PostResult>;
  updateMessage(options: UpdateMessageOptions): Promise<PostResult>;
  deleteMessage(channel: string, ts: string): Promise<void>;
  createChannel(name: string): Promise<ChannelInfo>;
  archiveChannel(channel: string): Promise<void>;
  inviteToChannel(channel: string, users: string[]): Promise<void>;
  // ... including history, files, reactions, and health checks
}

ISlackSocket

Abstraction for receiving real-time events over Slack Socket Mode.

interface ISlackSocket {
  connect(onMessage: MessageHandler): Promise<void>;
  disconnect(): Promise<void>;
  isConnected(): boolean;
}

type MessageHandler = (message: IncomingSlackMessage) => Promise<void>;

interface IncomingSlackMessage {
  channel: string;
  user: string;
  text: string;
  ts: string;
  thread_ts?: string;
}

ITmuxService

Manages interactions with local tmux terminal multiplexer sessions.

interface ITmuxService {
  hasSession(session: string): Promise<boolean>;
  ensureSession(session: string): Promise<void>;
  createWindow(session: string, window: string): Promise<void>;
  sendMessage(target: string, message: string): Promise<void>;
  killSession(session: string): Promise<void>;
  listSessions(): Promise<string[]>;
}

IRedisService

Queue interaction abstraction.

interface IRedisService {
  push(key: string, value: string): Promise<void>;
  bpop(key: string, timeout: number): Promise<string | null>;
  pop(key: string): Promise<string | null>;
  len(key: string): Promise<number>;
  del(key: string): Promise<void>;
  ping(): Promise<void>;
  quit(): Promise<void>;
}

Key Application Classes

SlackService

Facade for all Slack operations, wrapping both ISlackClient and ISlackSocket so the rest of the application has a unified interface.

ProjectService

Acts as a Repository. Abstracts the persistent project state JSON file, exposing CRUD methods to register, modify, and track active project spaces and configurations.

SlackListenerDaemon

Constantly listens to the socket. Looks up target projects based on the channelId of the incoming message, and dispatches the payload to the respective Redis queue.

MessageProcessorDaemon

Operates inside project-specific background jobs. Uses blocking pop (bpop) to read queued tasks from Redis, formats the data cleanly, and invokes TmuxService.sendMessage().
